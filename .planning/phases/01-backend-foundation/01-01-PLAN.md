<task type="auto">
  <name>Task 1: Create initial schema migration with all tables, RLS, triggers, and functions</name>
  <files>supabase/migrations/00001_initial_schema.sql</files>
  <action>
Create a single Supabase migration file that deploys the complete schema. Use `mcp__supabase__apply_migration` with name `initial_schema`. The SQL must include ALL of the following, in this exact order:

**1. Extensions:**
- `CREATE EXTENSION IF NOT EXISTS pg_trgm;` (for fuzzy text search on geo_locations)
- `CREATE EXTENSION IF NOT EXISTS btree_gist;` (for EXCLUDE constraint on user_locations)

**2. Helper functions (created before tables that reference them):**

- `update_updated_at()` -- generic trigger function that sets `NEW.updated_at = NOW()`
- `handle_new_user()` -- creates profiles row on auth.users INSERT. Extracts platform from `NEW.raw_user_meta_data->>'platform'` (default 'unknown'), email from `NEW.email`, display_name from `COALESCE(NEW.raw_user_meta_data->>'full_name', NEW.raw_user_meta_data->>'name')`. Sets `platforms = ARRAY[user_platform]`. Must be `SECURITY DEFINER`.
- `handle_new_profile()` -- creates user_settings row on profiles INSERT. Simply `INSERT INTO public.user_settings (user_id) VALUES (NEW.id)`. Must be `SECURITY DEFINER`.
- `increment_content_version()` -- on prayer_content UPDATE, sets `NEW.version = OLD.version + 1`
- `is_premium_user(check_user_id UUID) RETURNS BOOLEAN` -- checks if user has active purchase where `status = 'active'` AND (`expires_at IS NULL OR expires_at > NOW()`). Must be `SECURITY DEFINER STABLE`.
- `search_locations(query TEXT, max_results INTEGER DEFAULT 20)` -- splits query on comma, uses first part as city name ILIKE match, optional second part as country filter. Orders by exact prefix match first, then alphabetically. Returns `SETOF public.geo_locations`. Must be `STABLE`.

**3. Tables (exact schemas from MIGRATION_SPEC.md sections 1.1-1.6, 2.1, 2.2):**

- `profiles` -- UUID PK referencing auth.users(id) ON DELETE CASCADE. Columns: id, display_name, email, avatar_url, firebase_uid (UNIQUE), migrated_at, migration_source, platforms (TEXT[] DEFAULT '{}'), app_version, created_at, updated_at, last_seen_at
- `purchases` -- UUID PK with gen_random_uuid(). Columns: id, user_id (FK to profiles), product_id, purchase_type (CHECK: legacy_onetime/apple_subscription/google_subscription/promotional), billing_source (CHECK: google_play/app_store/promotional/legacy_migration), store_transaction_id, store_receipt, original_purchase_date, status (CHECK: active/expired/cancelled/refunded/revoked/paused, DEFAULT 'active'), expires_at, grace_period_end, cancellation_date, is_legacy_user (DEFAULT FALSE), legacy_access_token, legacy_purchase_date, created_at, updated_at, verified_at. UNIQUE constraint on (user_id, product_id, billing_source, store_transaction_id). Indexes on user_id and expires_at WHERE status='active'.
- `user_settings` -- UUID PK (user_id) referencing profiles. All columns per MIGRATION_SPEC.md section 1.3 including nusach, is_woman, language, is_in_israel, is_mizrochnik, mukaf_mode, date_change_rule, pasuk, sick_name, tal_preference, shabbat_candle_minutes, shabbat_end_minutes, dawn_opinion, sunrise_opinion, zman_opinion, dusk_opinion, settings_version, created_at, updated_at. Include all CHECK constraints exactly as specified.
- `user_locations` -- UUID PK with gen_random_uuid(). Columns per MIGRATION_SPEC.md section 1.4. Include EXCLUDE constraint for unique selected per user. Index on user_id.
- `geo_locations` -- TEXT PK (geoname_id). Columns per MIGRATION_SPEC.md section 1.5. Trigram GIN indexes on name and country_name. Standard indexes on country_code and (latitude, longitude).
- `user_notifications` -- UUID PK with gen_random_uuid(). Columns per MIGRATION_SPEC.md section 1.6 including notification_type CHECK, zman_type CHECK, sound_setting CHECK, active_days INTEGER[].
- `prayer_content` -- BIGINT IDENTITY PK. Columns per MIGRATION_SPEC.md section 2.1: content_key (UNIQUE), content_type (CHECK: prayer/title/instruction/ui), text_variants (JSONB), translations (JSONB), placeholders (TEXT[]), default_expand (CHECK), show_title, add_to_menu, sort_order, section, version, created_at, updated_at. Indexes on section, version, and GIN on text_variants.
- `content_versions` -- BIGINT IDENTITY PK. Create sequence `content_global_version_seq` first. Columns per MIGRATION_SPEC.md section 2.2: section, global_version (DEFAULT nextval), change_type (CHECK: create/update/delete), affected_keys (TEXT[]), description, created_at. Index on global_version.

**4. RLS policies (enable on ALL tables):**
- `profiles`: SELECT/UPDATE own row (auth.uid() = id). No INSERT policy needed (trigger uses SECURITY DEFINER).
- `purchases`: SELECT own rows (auth.uid() = user_id). ALL for service_role (auth.role() = 'service_role').
- `user_settings`: ALL own row (auth.uid() = user_id).
- `user_locations`: ALL own rows (auth.uid() = user_id).
- `geo_locations`: SELECT for anyone (TRUE).
- `user_notifications`: ALL own rows (auth.uid() = user_id).
- `prayer_content`: SELECT for anyone (TRUE). ALL for service_role.
- `content_versions`: SELECT for anyone (TRUE).

**5. Triggers:**
- `on_auth_user_created` -- AFTER INSERT ON auth.users -> handle_new_user()
- `on_profile_created` -- AFTER INSERT ON profiles -> handle_new_profile()
- `updated_at` triggers on: profiles, purchases, user_settings, user_locations, user_notifications (BEFORE UPDATE -> update_updated_at)
- `prayer_content_version` -- BEFORE UPDATE ON prayer_content -> increment_content_version()

**Important implementation notes:**
- Use `auth.uid()` not `current_user` for RLS checks
- Use `auth.role()` for service_role checks on purchases and prayer_content
- The `handle_new_user` function runs in auth schema context so needs SECURITY DEFINER to write to public.profiles
- Do NOT add updated_at trigger to geo_locations (read-only reference data) or content_versions (append-only log)
- Do NOT add updated_at trigger to prayer_content (it already has the version increment trigger; updated_at is handled by update_updated_at separately -- actually DO add the updated_at trigger to prayer_content as well, the version trigger handles version and updated_at trigger handles the timestamp)
