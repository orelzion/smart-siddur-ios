Build the complete settings system (synced + local) and location picker. Users can configure their halachic identity (nusach, zmanim opinions, personal insertions) with changes persisting to Supabase, configure display preferences (theme, font) locally with instant response, and search/select their location from 141K cities or auto-detect via GPS.

Purpose: Settings and location are prerequisites for zmanim calculation and prayer generation. Nusach determines prayer text variants. Location determines zmanim times. Opinions determine which zmanim calculations to use.

Output: Working settings screens for all synced and local preferences, location picker with search and GPS, all wired to Supabase backend.
</objective>

    **LocationRepository.swift**: Protocol + implementation:
    - `func searchLocations(query: String) async throws -> [GeoLocation]` -- calls `supabase.rpc("search_locations", params: ["query": query, "max_results": 20])`. Debounce on the view side (300ms).
    - `func findNearestCity(latitude: Double, longitude: Double) async throws -> GeoLocation?` -- queries geo_locations ordered by distance: `supabase.from("geo_locations").select().order("abs(latitude - \(lat)) + abs(longitude - \(lng))").limit(1)`. Or use a raw RPC/PostgREST computed column approach. Simplest: fetch 50 nearest by bounding box, sort client-side by Haversine distance.
    - `func getSelectedLocation() async throws -> UserLocation?` -- `supabase.from("user_locations").select().eq("user_id", value: userId).eq("is_selected", value: true).single()`
    - `func saveLocation(_ geo: GeoLocation, isFromGps: Bool) async throws -> UserLocation` -- Deselect current (update is_selected=false), then insert new user_location with is_selected=true. Include geoname_id, all geo fields, and timezone. Handle the EXCLUDE constraint by deselecting first.
    - `func getUserLocations() async throws -> [UserLocation]` -- fetch all saved locations for the user

    **LocationViewModel.swift**: `@Observable` class:
    - `searchText: String` (bound to search field)
    - `searchResults: [GeoLocation]` (updated on search)
    - `selectedLocation: UserLocation?` (current active location)
    - `isSearching: Bool`, `isDetectingGPS: Bool`
    - `gpsStatus: GPSStatus` (enum: notDetermined, authorized, denied, detecting, found, error)
    - On searchText change (debounced 300ms): call searchLocations
    - `detectGPSLocation()`: request CLLocationManager permission, get current coordinate, call findNearestCity, show result with distance indicator (e.g., "Jerusalem (2km away)")
    - `selectLocation(_ geo: GeoLocation, isFromGps: Bool)`: save to user_locations via repository
    - On init: load current selected location
