  <action>
    **AuthRepository.swift**: Protocol + implementation wrapping Supabase Auth:
    - `signInWithApple()` -- uses `supabase.auth.signInWithApple()` (ASAuthorizationAppleIDProvider flow). Supabase Swift SDK has built-in Apple Sign-In support.
    - `signInWithGoogle()` -- uses GoogleSignIn SDK to get ID token, then `supabase.auth.signInWithIdToken(credentials: .init(provider: .google, idToken: idToken))`.
    - `signInAnonymously()` -- uses `supabase.auth.signInAnonymously()`.
    - `signOut()` -- uses `supabase.auth.signOut()`.
    - `currentSession` -- published property from `supabase.auth.session`.
    - `authStateChanges` -- async stream from `supabase.auth.authStateChanges`.
    Session persistence is handled automatically by supabase-swift (stores in Keychain by default).

    **AuthViewModel.swift**: `@Observable` class.
    - Properties: `isAuthenticated: Bool`, `isLoading: Bool`, `error: String?`, `currentUser: UserProfile?`
    - Methods: `signInWithApple()`, `signInWithGoogle()`, `signInAnonymously()`, `signOut()`
    - On init, subscribe to `authStateChanges` to update `isAuthenticated`.
    - Error handling: catch and expose user-friendly messages.

    **LoginView.swift**: Full-screen login with:
    - App logo/title at top
    - "Sign in with Apple" button (using SignInWithAppleButton from AuthenticationServices)
    - "Sign in with Google" button (using GIDSignInButton or custom styled button)
    - "Continue without account" text button for anonymous auth
    - Loading overlay during auth operations
    - Error alert for failures

    **OnboardingView.swift**: Simple wrapper that shows LoginView. Placeholder for future onboarding steps (nusach selection, location setup). For now, just shows LoginView.

    **TabContainerView.swift**: Post-auth container with TabView:
    - Tab 1: Home (placeholder ZmanimView -- just Text("Zmanim") for now)
    - Tab 2: Calendar (placeholder -- Text("Calendar"))
    - Tab 3: Settings (placeholder -- Text("Settings"))
    Uses system SF Symbols for tab icons.

    **HomeView.swift**: Placeholder that shows user's display name/email and a sign-out button. Proves auth is working.

    **SmartSiddurApp.swift** (update): Wire auth state observation. If session exists, show TabContainerView. If no session, show OnboardingView.

    Important implementation notes:
    - Apple Sign-In: Supabase Swift SDK provides `signInWithApple()` that handles the full ASAuthorization flow internally. No need to manually build the Apple auth request.
    - Google Sign-In: Use `GIDSignIn.sharedInstance.signIn(withPresenting:)` to get tokens, then pass `idToken` to Supabase `signInWithIdToken`.
    - Anonymous: Simplest -- just `supabase.auth.signInAnonymously()`. User gets a real UUID and can link an identity later.
    - The `handle_new_user` trigger on the backend auto-creates a `profiles` row and subsequently a `user_settings` row -- no client-side setup needed.
