## Decisions Made

1. **XcodeGen info block for Info.plist** -- Switched from `GENERATE_INFOPLIST_FILE: YES` with `INFOPLIST_KEY_*` build settings to XcodeGen's `info.properties` block. This allows custom keys (GIDClientID, GIDServerClientID, CFBundleURLTypes) that cannot be set via INFOPLIST_KEY build settings.

2. **GIDServerClientID for token exchange** -- Set `GIDServerClientID` to the web client ID (not iOS client ID) so the Google ID token audience matches what Supabase expects for `signInWithIdToken`.

3. **AuthRepository protocol pattern** -- Defined `AuthRepositoryProtocol` for testability. Future tests can mock auth without hitting Supabase.

4. **Auth state via async stream** -- Used `supabase.auth.authStateChanges` async stream at the app root level to reactively switch between login and authenticated views.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Missing GIDClientID in Info.plist caused Google Sign-In crash**
- **Found during:** Task 3 (Checkpoint -- user verification)
- **Issue:** Google Sign-In SDK requires `GIDClientID` in Info.plist. The original implementation used `GENERATE_INFOPLIST_FILE: YES` which doesn't support custom keys like `GIDClientID`.
- **Fix:** Switched to XcodeGen `info` block to generate Info.plist with `GIDClientID`, `GIDServerClientID`, and reversed client ID URL scheme. Also wired `onOpenURL` to forward URLs to `GIDSignIn.sharedInstance.handle(url)`.
- **Files modified:** project.yml, Info.plist (new), SmartSiddurApp.swift, SmartSiddur.xcodeproj/project.pbxproj
- **Verification:** Build succeeded, Info.plist contains all three required values
- **Committed in:** `306e26b`

## Issues Encountered

- Google Sign-In SDK crashes at runtime if `GIDClientID` is missing from Info.plist -- no compile-time warning. This is a known SDK behavior where configuration is read from Info.plist at runtime.
- `GENERATE_INFOPLIST_FILE: YES` (Xcode's auto-generated Info.plist) does not support arbitrary custom keys like `GIDClientID`. Had to switch to XcodeGen-managed Info.plist generation via the `info` block.
